Crea un formulario para dejar un comentario en una página: Usuario y comentario de texto

Este reto conlleva conocer 3 etiquetas, form, input y textarea

  • form: Etiqueta contenedora del formulario
  • input: Etiqueta para obtener datos del usuario
  • texarea: Igual que input, pero se utiliza para escribir textos extensos

FORM

Un formulario es el mecanismo que utiliza un usuario para enviar información. Tiene los siguientes atributos

  • action: Dirección URL del backend al cuál se enviará la información obtenida en el formulario.
  • method: Método HTTP de envío. GET a través de URL, POST para envío extenso de información.
  • name: Nombre del formulario. Útil para identificar el formulario y procesar posteriormente.
  • target: Nombre del lugar donde se abrirá el formulario. \_blank para nueva pestaña.
  • enctype: Codificación para el envío del formulario. Importante para envío de archivos.
  • accept-charset: Fuerza a utilizar una codificación en los parámetros de texto del formulario.
  • autocomplete: Nombre del lugar donde se abrirá el formulario. \_blank para nueva pestaña.
  • novalidate: Desactiva la validación HTML5 del formulario.

Esto solo nos indica como es el contendor del formulario, para que el usuario pueda introducir los datos se utlizan las siguientes etiquetas:

  • <input>: Información de texto corto. Nombres, apellidos, textos cortos...
  • <textarea>: Información de texto largo. Comentarios, observaciones...
  • <input type="number"> ó <input type="range">: Números o cantidades. Edades, precios, cuantías..
  • <input type="date"> o derivados: Fechas u horas. Fecha de nacimiento, inicio de evento...
  • <input type="checkbox">: Verdadero/falso. Si/No, Opción A/B, ON/OFF...
  • <input type="radio"> ó <select>: Opción única (Elegir una opción de 2 ó más posibles)
  • <select multiple> ó <input type="checkbox">: Varias opciones (Elegir varias opciones de 2 ó más posibles)
  • <datalist>: Opción única abierta (Elegir una opción o indicar una propia)
  • <input type="color">: Selección de color (Escoger un color o tonalidad)
  • <input type="file">: Selección de archivo (Escoger un archivo para enviar)

Con lo que un formulario básico quedaría de la siguiente forma:

            
                <form name="userdata" method="post" action="/send/">
                    <input type="text" name="username">
                    <input type="textarea" name="comment">
                    <input type="submit">
                </form>

Este formulario utilizará el método post (envío de información "oculta") a la dirección dominio_utilizado/send y tiene el nombre de userdata para poder procesarlo.

El input de tipo texto lo utilizamos para el nombre del usuario

El input de tipo texarea lo utilizamos para introducir el comentario

El input de tipo submit sirve para enviar la información, funciona como un botón

INPUT

La etiqueta input no tiene etiqueta de cierre y tiene muchos atributos:

  • name: Indica el nombre del campo. Útil para hacer referencia a la información almacenada.
  • type: Indica la naturaleza de datos que se espera del usuario (texto, número, fechas, etc...).
  • value: Indica el valor inicial que tendrá este campo de datos. Si se omite, está vacío.
  • form: Asocia este campo con un formulario específico. Útil si necesitamos tenerlo fuera del <form>.
  • list: En algunos <input> permite combinar con otra etiqueta para personalizarla.
  • placeholder: Indica una sugerencia al usuario antes de escribir el valor en el campo.
  • size: Tamaño visual (aprox: número de carácteres) del campo de datos.
  • autocomplete: Activa on o desactiva off el autocompletado para este campo. Activado por defecto.
  • autofocus: Establece el foco (coloca el cursor) en este campo al cargar la página.

A su vez el atributo type puede tener los siguientes valores para indicar el tipo de dato a introducir

  • text: Texto alfanumérico. (Por defecto)
  • number: Solo permite números
  • search: Texto para búsquedas
  • tel: Números de teléfono
  • url: Direcciones urls
  • mail: Emails
  • password: Oculta en la PANTALLA lo que se escribe.
  • hidden: No se muestra este input

El tipo de input además de permitir al navegador hacer pequeñas validaciones, cambia el teclado que muestra si se rellena desde un móvil/tablet.

TEXTAREA

Hay tres atributos a destacar:

  • cols:Número de carácteres que caben en horizontal (columnas).
  • rows: Número de carácteres que caben en vertical (filas).
  • wrap: Con soft (por defecto) o hard puedes indicar si el texto debe enviarse ajustado o tal cuál se escribió.

Se puede indicar el tamaño del texarea desde HTML o CSS (con width y heigth) e incluso en ocasiones interesará hacerlo con ambos

Wrap nos permite decidir como enviar la información al servidor.

  • soft: Enviará el texto literalmente (por defecto)
  • hard: Ajustará el texto como se ve exactamente en el textarea y añadirá saltos de líneas (\n)

Si el texto es demasiado grande, aparecerán barras de scroll, pero esto se puede evitar con la propiedad CSS field-sizing

  • fixed: Tamaño fijo (valor por defecto)
  • content: Tamaño crece a medida que el contenido aumenta

Por último, es interesante la propiedad CSS resize: none que impide al usuario ampliar el área de texto
