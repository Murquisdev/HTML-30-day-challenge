Las etiquetas en HTML tienen diferentes usos, a nivel de documentos, agrupaciones, formularios, tablas...

Hay una serie de etiquetas HTML que se consideran de tipo semánticas

Estas etiquetas semánticas existen a nivel de texto y a nivel de agrupaciones y sirven para dar contexto a los buscadores y al utilizarlas conseguiremos mejorar el posicionamiento.


Por otro lado a la hora de "maquetar" la web se suele utilizar solo la etiqueta div creando algo como el ejemplo siguiente:

        
            <div class="articulo">
                <h1>Título del artículo</h1>
                <p class="introduccion">Pequeña introducción.</p>
                <p class="contenido">Aquí va el texto del artículo en cuestión con sus detalles.</p>
                <p class="pie">Escrito por Manz</p>
            </div>

Este ejemplo cumple con los requisitos que necesitamos, pero no es semánticamente correcto, con lo que se podría sustituir de la siguiente forma:

        
            <article>
                <header>
                  <h1>Título del artículo</h1>
                  <p class="abstract">Pequeña introducción.</p>
                </header>
                <p class="content">Aquí va el texto del artículo en cuestión con sus detalles.</p>
                <footer>
                  <p>Escrito por Manz.</p>
                </footer>
            </article>

Aquí indicamos que es un artículo independiente, tiene un título, un contenido y pie, separando todo semánticamente

- ¡CUIDADO! Aunque existe la etiqueta main para indicar el contenido principal, solo debe aparecer UNA VEZ en todo el documento
